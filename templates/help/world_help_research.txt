Research Project Creation
==========================

Academic research projects with data management (Constitution §III.V)
Updated: 2025-12-10 (Constitution v1.0.17)

Purpose
-------

Creates standardized research project with data analysis workflow. Follows project-by-activity principle with co-located provenance for reproducible research.

Usage
-----

  dirforge init research [options]
  dirforge init research --title "Project Name"

Required Options
----------------

  --title "Project Title"                       Human-readable project title (converted to PROJECT-ID)

Optional Options
----------------

  --python 3.11                                 Python version for conda environment (default: 3.11)
  --no-conda                                    Skip conda environment creation
  --conda-packages "pkg1 pkg2"                  Additional packages beyond base set

Naming Conventions
------------------

  Project ID Format:        YYYY_<snake_case_title>
  Example Input:            Thermal Model Analysis
  Generated ID:             2025_thermal_model_analysis
  Conda Environment:        research_thermal_model_analysis
  Rules:                    ASCII lowercase [a-z0-9_-] only

Directory Structure
-------------------

  RESEARCH_WORLD/2025_project_name/
  ├── 00_admin/                   # contracts, ethics, agreements
  ├── 01_project_management/      # proposals, budgets, reports, presentations
  │   ├── README.md
  │   ├── 01_proposal/
  │   │   ├── 01_draft/
  │   │   ├── 02_submission/
  │   │   ├── 03_review/
  │   │   └── 04_final/
  │   ├── 02_finance/
  │   ├── 03_reports/
  │   └── 04_presentations/       # kept flat for easy previews
  ├── 02_studies/                 # per-study workspaces (study-based organization)
  ├── .integrity/
  │   ├── checksums/
  │   └── manifests/
  ├── README.md
  └── project.yaml

Project Management (01_project_management)
------------------------------------------

The `01_project_management/` folder centralizes project-level management artifacts:

- `01_proposal/`: proposal lifecycle folders split into `01_draft/`, `02_submission/`, `03_review/`, and `04_final/`.
- `02_finance/`: budgets, grant letters, invoices and financial reports.
- `03_reports/`: periodic reports and final deliverables.
- `04_presentations/`: slides, posters and presentation assets (by convention kept flat to simplify previews and exports).

Use the README files rendered into each subfolder to document purpose and naming conventions. The scaffolder renders templates with `{{PROJECT_NAME}}` and `{{PROJECT_ID}}` placeholders.

Examples
--------

Common Usage:

  # Interactive mode with prompts
  dirforge init research
  # Prompts for: Project title
  
  # Direct specification
  dirforge init research --title "Thermal Analysis"
  
  # Custom Python version
  dirforge init research --title "ML Study" --python 3.12
  
  # Skip conda environment (theory-only)
  dirforge init research --title "Theory Work" --no-conda
  
  # Preview structure without creating
  dirforge --dry-run init research --title "Test Project"
  
  # Backup existing project before overwrite
  dirforge init research --title "Existing" --backup

Best Practices
--------------

  • Use descriptive PROJECT-ID that will make sense in 5 years
  • Include year prefix for chronological organization
  • Place large datasets in external storage with manifests in 04_data/
  • Use .integrity/checksums/ for data validation
  • Version control 05_data_analysis/ with Git
  • Archive final results in 06_data_outputs/ with version tags
